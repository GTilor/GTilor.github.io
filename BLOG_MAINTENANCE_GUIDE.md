# 📘 GTilor 的博客维护手册

这份指南将告诉你如何发文章、管理评论，以及你的博客是怎么“跑”起来的。

---

## 1. 简易原理说明（为什么要搞这么复杂？）

你可以把这几家公司的服务看作是你的“数字房产”：

*   **GitHub (地基)**：存放你博客的所有原始代码和发布的网页文件。
*   **Vercel (管家)**：它是评论区的小服务器。它平时在那儿待命，当有人评论时，它会出面接住评论，并存进仓库。
*   **MongoDB (仓库)**：专门存评论的地方。
*   **Hexo (装修工具)**：就在你的电脑上。你写的是普通的 Markdown 文本，它负责把它们“刷”成漂亮的 HTML 网页。

---

## 2. 如何写一篇新文章

写文章的流程就像是“在电脑上写好 -> 预览 -> 推送到全世界”。

### 第一步：创建文件
在终端（电脑那个黑框）里输入：
```bash
cd /Users/liyuguo/Projects/web/my-blog
hexo new "我的新文章标题"
```
终端会告诉你文件创建在了 `source/_posts/` 文件夹里。

### 第二步：编辑内容
1.  去文件夹里找到那个 `.md` 文件，用你喜欢的编辑器（比如 VS Code）打开。
2.  你会看到开头有一段 `---` 包围的文字，那是**标题、日期、标签**，改好之后在下面直接写正文。

### 第三步：本地预览
在终端输入：
```bash
hexo s
```
打开浏览器访问 `http://localhost:4000` 看看文章好不好看。如果满意，按 `Ctrl+C` 停止。

---

## 3. 如何发布上线

当你写好了文章，或者像今天一样改好了配置，输入这三行暗号：

```bash
hexo clean  # 打扫一下旧文件
hexo g      # 生成（Generate）新网页
hexo d      # 部署（Deploy）到 GitHub 上
```
执行完 `hexo d` 后，等一两分钟，刷新你的域名就能看到了。

---

## 4. 如何管理留言区

你的评论区（Twikoo）非常智能，管理它**不需要**再去写代码。

### 获取管理员身份
1.  在你的**正式博客网页**上，找到评论区。
2.  你会看到评论框右下角有一个**蓝色的小齿轮**图标。
3.  第一次点击它，它会让你**设置一个管理员密码**。设置好并登录。

### 日常管理
登录后，你可以：
*   **删除评论**：看到垃圾广告直接点删除。
*   **审核评论**：可以设置必须你通过才显示的评论。
*   **回复提醒**：你甚至可以配置当有人回复你时，给你发个邮件。

---

## 5. 必须注意的安全事项

> [!CAUTION]
> 1. **不要泄露密码**：你在 Vercel 和 MongoDB 里填的那个数据库密码是死命令，不要发到朋友圈或者公开群，也不要写在本文档等明文代码中。
> 2. **环境变量**：Vercel 里的 `MONGODB_URI` 只有在你重新部署（Redeploy）时才会更新，改了它记得去点一下重新部署。
